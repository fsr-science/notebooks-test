Step 5 of 7

Vercel Deployment Setup

Follow these steps to deploy the backend API. The installer will complete the final wiring once you're done.

1
Create a Fine-Grained Personal Access Token Go to GitHub → Settings → Developer Settings → Fine-grained tokens → Generate new token. Set Expiration to No expiration. Under Repository access choose your repo, then grant:

• Contents — Read & Write
• Actions — Read & Write

Copy the token — you will paste it as WM_PAT in Vercel.
2
Create a free Vercel Hobby account Sign up at vercel.com/signup and connect your GitHub account.
3
Import your repository & create a new project On your Vercel dashboard click Add New → Project, then select your repository.

�� The project name becomes your public URL: project-name.vercel.app

Before deploying, add these Environment Variables:
NameValue
WM_PATThe Fine-Grained token from step 1
WM_SATOKENThe 6-char code you choose below — your super-admin login
4
Click Deploy — Vercel will build and publish the project. Keep the deployment URL handy for the next step.
Exactly 6 characters — letters and digits only.
You will add this as the WM_SATOKEN environment variable in Vercel. It becomes your super-admin login code inside the app.
